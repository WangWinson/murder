    Here is an overview of the Murder codebase:

    • docs/ — Target directory for generated documentation. Everything that follows, is a result of what you see here.
    • example/ — Example use case for Murder.
      • example/author.js — Author ID generator for client and server.
      • example/client.js — Basic client example script, used as bundle entry point with browserify.
      • example/config.js — Basic client/server config.
      • example/Crow.js — Model sub-class represents a crow.
      • example/Murder.js — Collection sub-class for a Murder of crows.
      • package.json — Example uses seperate dependencies.
      • example/public/ — Public HTTP file server directory.
        • example/public/index.html — Main HTML file.
        • example/public/main.css — Main CSS file.
      • example/relays.js — Common relays definition script.
      • example/server.js — Basic server example script.
      • example/sources.js — Common sources definition script.
      • example/tessel.js — Basic client on a tessel.
    • gulpfile.js — Useful gulp tasks for development.
    • karma/ —
    • lib/ — Core CRDT implementation plus extra adapters and types.
      • lib/core/ — Core CRDT implementation plus extra adapters and types.
        • lib/core/ConflictFreeReplicatedDataType.js — CRDT base class.
        • lib/core/Operation.js — CRDT operation class.
        • lib/core/Relay.js — Source message relay class.
        • lib/core/Source.js — CRDT source base class.
      • lib/debug.js — Debug logger.
      • lib/extras/ — Extra integration resources.
        • lib/extras/ReactMixin.js — Easy React integration.
      • lib/index.js — Default module entry point. This file.
      • lib/relays/ — Different Relay implementations.
        • lib/relays/HTTPSource.js — HTTP Relay implementation.
        • lib/relays/StompSource.js — Stomp Relay implementation.
        • lib/relays/WebRTCSource.js — WebRTC Relay implementation.
        • lib/relays/WebSocketSource.js — WebSocket Relay implementation.
      • lib/sources/ — Different Source implementations.
        • lib/sources/NedbSource.js — NeDB Source adapter.
        • lib/sources/RedisSource.js — Redis Source adapter.
        • lib/sources/WebStorageSource.js — WebStorage Source adapter.
      • lib/types/ — Various CRDT data structures.
        • lib/types/Collection.js — A list of other CRDT objects.
        • lib/types/Graph.js — A graph of other CRDT objects.
        • lib/types/Model.js — A basic CRDT document object.
        • lib/types/Text.js — A text CRDT for collaboration.
    • LICENSE — Modified copyleft ISC license. It is in every file. See bottom.
    • package.json — Core module dependencies and NPM details.
    • README.md — Read this file first.
    • test/ — Test scripts. Omitted from docs.
